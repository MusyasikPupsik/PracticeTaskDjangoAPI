# Django REST API - Полная реализация ролевой модели доступа

Проект полностью доработан и реализует разграничение доступа на уровне отдельных объектов (Object-level permissions) на основе предоставленной структуры таблиц (`ProjectMembership`, `TeamMembership` и `Curator`).

## Реализованная ролевая модель:
1. **Просмотр (`viewer`)**: Пользователь имеет доступ исключительно к безопасным методам чтения (`GET`, `HEAD`, `OPTIONS`). Любые попытки изменения данных (`POST`, `PUT`, `PATCH`, `DELETE`) блокируются системой с ответом `403 Forbidden`.
2. **Редактирование (`editor` / `owner`)**: Пользователи с данными ролями имеют полный доступ ко всем CRUD-операциям в рамках своих проектов.
3. **Кураторы (`Curator`)**: Обладают глобальными административными правами доступа ко всем веткам данных.

## Как протестировать ролевую схему локально:
1. Создайте объект проекта в панели администратора по адресу: http://localhost:8000/admin/
2. Привяжите вашего пользователя к созданному проекту через таблицу **Project memberships** и выберите роль `viewer`.
3. Перейдите по ссылке детального просмотра инстанса проекта: http://localhost:8000/api/projects/1/
4. Попробуйте выполнить запрос `PUT`, `PATCH` или `DELETE` через форму внизу страницы. Система выдаст ошибку: `"You do not have permission to perform this action."`
