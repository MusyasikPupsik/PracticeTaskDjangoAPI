# Django REST API - Полная реализация системы гипотез и ролевого доступа

Финальное задание по производственной практике. Архитектура проекта полностью доработана в соответствии со всеми 23 моделями структуры базы данных и ролевой схемы.

## Реализованный функционал
* **Полное покрытие моделей**: Созданы сериализаторы (`serializers.py`) и контроллеры (`views.py`) для всех 23 сущностей (включая `Action`, `Data`, `Insight`, `AssistantFunction`, `ProjectMembership`, `TeamMembership` и др.).
* **Ограничение доступа (Permissions)**: Все эндпоинты защищены с помощью `IsAuthenticated`. Доступ к данным предоставляется только авторизованным пользователям.
* **JWT-Аутентификация**: Интегрирована библиотека `django-rest-framework-simplejwt`. Фронтенд может запрашивать access- и refresh-токены для авторизации пользователей.

## Основные эндпоинты API
* `/api/users/` — Управление пользователями
* `/api/projects/` — Управление проектами
* `/api/hypotheses/` — Управление гипотезами
* `/api/auth/token/` — Получение JWT-токена (вход для фронтенда)
* `/api/auth/token/refresh/` — Обновление JWT-токена

## Инструкция по локальному запуску

1. Установите все необходимые зависимости из манифеста:
```bash
pip install -r requirements.txt
```

2. Выполните миграции базы данных:
```bash
python manage.py migrate
```

3. Создайте суперпользователя для авторизации:
```bash
python manage.py createsuperuser
```

4. Запустите локальный сервер:
```bash
python manage.py runserver
```

* Панель администратора: http://127.0.0
* Полный интерактивный интерфейс API: http://127.0.0
